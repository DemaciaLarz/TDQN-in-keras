Buy and Hold¶

This means that you obtain a fully exposed position to the market at all time. You buy and sit on it. In effect this means that you follow the underlying assets movements.

Sharpe Ratio¶

The sharpe ratio compares the investment with a risk free asset.

Ok, so it looks exactly like the actual curve, but shifted downwards. Maybe we get back to this and compare it to some indexes or what not.

Lets compare with the gpu3_9249 and random to get some perspective.

Usually, any Sharpe ratio greater than 1.0 is considered acceptable to good by investors.

  • A ratio higher than 2.0 is rated as very good.
  • A ratio of 3.0 or higher is considered excellent.
  • A ratio under 1.0 is considered sub-optimal.

Information Ratio¶

It is similar to sharpe ratio but instead of using a risk free return as benchmark, it uses a risky index, such as the S&P500 or the OMX Stockholm 30 Index (OMXS30) which we use here.

Benchmarks¶

Let the curves below represent the official metrics that we will want to outperform with upcoming model improvements, that is:

  • Log returns: increase the area between model and asset.
  • Sharpe ratio: with respect to zero as riskfree asset. Sure, we can use this aswell.
  • information ratio: this one feels like the truly useful one. Here we do not only see how the model stands with respect to the asset it is trained on, but on other real assets as well. Lets run coming improvements against omx to start with, then we can fill it up with more.

def get_log(data):
    '''Computes the log returns of given data.'''
    log_r = []
    count = 0
    for i in data:
        if i >= 0:
            count = count + np.log(1 + abs(i) + 0.000001)
        if i < 0:
            count = count - np.log(1 + abs(i) + 0.000001)
        log_r.append(count)
    return log_r
